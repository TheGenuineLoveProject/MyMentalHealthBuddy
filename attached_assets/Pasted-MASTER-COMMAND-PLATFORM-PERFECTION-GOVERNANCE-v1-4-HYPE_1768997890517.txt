AGENT 1 — QA GUARDIAN (LOGIN + ADMIN + DETERMINISM)
Goal: deterministic tests; no ECONNREFUSED/EADDRINUSE; login/admin behavior correct and safe.

1) Inspect:

2) Fix root causes of flakiness:
Implement MINIMUM deterministic harness so tests never require a manually running dev server.
Preferred:

  a) spawn server as child process
  b) wait for `/api/health` with bounded polling + timeout
  c) run vitest

Alternative:
- vitest `beforeAll/afterAll` for auth tests only to spawn/kill server.

3) Auth safety/status consistency (only if required by tests):
- Missing required fields → 400 (generic safe message)
- Invalid credentials OR rate-limited login → 401 “Invalid credentials” (uniform; never reveal rate limiting)
- Unauthenticated access → 401
- Authenticated non-admin hitting admin routes → 403
- Never leak stack traces/DB errors/internal IDs in responses

4) Admin readiness (no scope creep):

- Add a minimal dev/test-only seed guarded by env var (e.g., TEST_SEED_ADMIN=1) OR deterministic test fixture.
- No new dependencies. No plaintext secrets. Use existing hashing/session patterns.

5) Verify until green:
- `npm run test:auth || npx vitest run tests/auth.test.mjs`
