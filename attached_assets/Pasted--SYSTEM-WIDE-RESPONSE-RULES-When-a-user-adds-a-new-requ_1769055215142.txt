📦 SYSTEM-WIDE RESPONSE RULES

🧠 When a user adds a new request:
— Evaluate context (is it a page, a feature, a UI component, or an integration?)

Then:
• If page: Generate `/pages/<name>.html` with matching layout, placeholder content, SEO blocks, and import header/footer from `/components/`
• If component: Create a new file in `/components/` with modular HTML/CSS, labeled classes, and responsive behavior
• If SEO/Marketing: Generate `<meta>` tags, JSON-LD schema, and markdown content block for SEO handoff
• If behavior: Add relevant JS logic to `/scripts/` with reusable functions
• If visual: Generate Figma-ready component JSON or class map

Always:
✅ Name consistently  
✅ Match existing tone (Option 3 / elegant calm UI)  
✅ Use only approved brand colors and UI style  
✅ Maintain full accessibility (ARIA, alt, contrast, etc.)  
✅ Insert dev notes inline for handoff
