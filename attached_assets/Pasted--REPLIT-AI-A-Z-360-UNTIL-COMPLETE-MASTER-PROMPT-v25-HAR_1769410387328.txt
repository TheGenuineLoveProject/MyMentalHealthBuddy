--------------------------------------------------------------------------------
2A) scan-duplicates.mjs (requirements)
--------------------------------------------------------------------------------
Must:
1) Hash files:
- Text normalization for JS/TS/TSX/JSX/JSON/MD/CSS
- Binary hash for images
2) Detect:
- exact duplicates (same hash)
- near duplicates (token similarity >= 0.90)
- name duplicates (same basename)
- shadow copies in folders: backup, old, copy, tmp, dist, build, out, .next, coverage
3) Create risk score:
+5 in client/src or server/src
+5 imported/referenced
+4 core files: App.tsx, routes, schema, auth, stripe, openai, db
+2 in backup-ish folders
-5 in dist/build/node_modules
4) Write:
- /docs/duplicates/scan-latest.md
- /docs/duplicates/scan-latest.json
- include “Top 50 highest-risk duplicate clusters”

--------------------------------------------------------------------------------
2B) scan-collisions.mjs (requirements)
--------------------------------------------------------------------------------
Goal: detect “duplicate work” even when files aren’t duplicates.

It must detect:
- Multiple implementations of the same endpoint path
- Multiple auth middlewares/strategies active simultaneously
- Multiple Stripe webhook handlers
- Multiple OpenAI client wrappers
- Multiple “schema” definitions
- Multiple “admin dashboards”

How:
- Parse server route registration (regex ok)
- Extract endpoint strings ("/api/...", "/health", "/webhooks/stripe")
- Build map: endpoint -> files
- Flag collisions when same endpoint appears in >1 file
- Output:
  /docs/duplicates/collisions-latest.md
  /docs/duplicates/collisions-latest.json

--------------------------------------------------------------------------------
2C) scan-architecture.mjs (requirements)
--------------------------------------------------------------------------------
Generate registry maps automatically:
- endpoints.md
- ui-routes.md (React Router routes)
- db-schema.md (Drizzle schema tables)
Write also to /docs/reports/latest.md under “Architecture snapshot”.

2D) dedupe-plan + apply-dedupe-safe
--------------------------------------------------------------------------------
dedupe-plan.mjs:
- reads scan-latest.json + collisions-latest.json
- writes /docs/duplicates/decisions.md as PROPOSED keepers & redirect plan

apply-dedupe-safe.mjs:

- NEVER delete
- write /docs/duplicates/quarantine-log.md
