PUBLIC SURFACE ALIGNMENT v2.2 — SITEWIDE LINK + BUTTON + ROUTE AUDIT (REPLIT-SAFE)

HARD RULES:
- ADDITIVE ONLY. Do not rename/delete/move/refactor existing files.
- Do not “clean up” legacy directories. Do not remove pages.
- No background autonomy. Human-trigger only.
- Any auto-fix must be deterministic and limited to safe cases (e.g., obvious broken hrefs).
- Must produce a single canonical report and a PASS/FAIL checklist.
- Must end with VERIFIED via npm run verify.

GOAL:
1) Detect broken navigation, dead links, missing routes, and non-functional buttons across the public surface.
2) Generate an actionable fix plan ordered by impact.
3) (Optional) Apply only safe fixes automatically; everything else produces exact file+line instructions.

SCOPE:
- Client routes (Wouter)
- Navbar/menus/footer links
- CTA buttons (pricing/newsletter/blog/tools/community/login)
- Admin nav links (admin pages)
- Server routes mounted and responding
- Redirect map sanity (531+ redirects)

PHASE 0 — BASELINE VERIFY (NO CHANGES)
Run:
- npm run verify
If FAIL, STOP and report the first failing step.

PHASE 1 — CREATE AUDIT SCRIPTS (READ-ONLY)
Create:
1) scripts/audit/public-surface-audit.mjs
   - Scans:
     - client/src/** for:
       - <a href="...">
       - <Link href="..."> (wouter)
       - navigate("...") / setLocation("...")
       - button onClick={() => ...navigate...}
     - Extracts all internal paths:
       - '/', '/pricing', '/newsletter', '/blog', '/tools', '/community', '/login', '/dashboard', etc.
   - Normalizes:
     - strips domain
     - removes trailing slashes (except '/')
   - Produces:
     - docs/reports/public-surface-audit.json
     - docs/reports/public-surface-audit.md

2) scripts/audit/route-snapshot.mjs
   - Reads server/index.mjs (and dev/app entrypoints if present) to infer mounted routers
   - Lists expected /api routes + key public routes
   - Produces docs/reports/route-snapshot.md

PHASE 2 — LIVE ROUTE PROBES (SAFE, LOCAL)
Create:
scripts/audit/probe-routes.mjs
- Starts from a known route list:
  - public pages (top 25)
  - health endpoints
  - top APIs (auth, mood, journal, ai chat history, billing checkout endpoint existence)
- Probes local server base:
  - default http://localhost:5000
- For pages:
  - accept 200/302 as pass
- For auth-required APIs:
  - accept 401 as pass (expected when not logged in)
- Produces:
  docs/reports/probe-routes.md with PASS/FAIL per route

PHASE 3 — BROKEN LINK CLASSIFICATION
In public-surface-audit.mjs classify each path:
A) OK — route exists (seen in router or probes)
B) MISSING_ROUTE — path referenced but no route/page found
C) STALE_REDIRECT — redirects to missing target
D) BUTTON_NO_ACTION — button looks like CTA but has no onClick/href
E) PLACEHOLDER — '#', 'javascript:void(0)', empty href

PHASE 4 — AUTO-FIX (OPTIONAL, SAFE ONLY)
Create:
scripts/audit/apply-safe-surface-fixes.mjs
SAFE FIXES ONLY:
- Replace href="#" used as placeholder with a TODO comment + disable pointer events + aria-disabled
- Replace empty internal hrefs with '/':
  - only if it’s clearly a logo/home link
- Add rel="noopener noreferrer" to external target="_blank"
- Add aria-label if icon-only link missing label (best-effort)
DO NOT:
- Invent new routes
- Create new pages without instruction
- Change existing route architecture

PHASE 5 — HUMAN FIX PLAN (THE MAIN OUTPUT)
Create docs/reports/PUBLIC_SURFACE_FIX_PLAN.md containing:
- Top 20 broken items by impact (Navbar/Footer/Primary CTAs first)
For each item include:
- Type (MISSING_ROUTE / BUTTON_NO_ACTION / PLACEHOLDER / STALE_REDIRECT)
- Exact file path(s)
- Exact code snippet to change (copy-paste)
- Suggested correct destination:
  - Prefer canonical pages you already have:
    /pricing /newsletter /blog /tools /community /crisis /login /dashboard
- PASS criteria for each fix

PHASE 6 — COMMANDS + VERIFICATION
Add npm scripts (package.json) if you already have a scripts section; otherwise create docs with commands only.
Commands required to run:
- node scripts/audit/public-surface-audit.mjs
- node scripts/audit/route-snapshot.mjs
- node scripts/audit/probe-routes.mjs
- (optional) node scripts/audit/apply-safe-surface-fixes.mjs
- npm run verify

FINAL REPORT REQUIRED:
A) All files created (full paths)
B) Summary counts:
   - total links scanned
   - broken by category
   - top 10 broken by impact
C) The fix plan file path
D) VERIFIED line with npm run verify result
