OPERATING LOOP (EVIDENCE → FIX → VERIFY → REPORT)
Run phases in order. If a phase fails, fix only that scope, then rerun that phase.

PHASE 1 — BASELINE EVIDENCE (MUST RUN + CAPTURE)
1) Environment:
   - `node -v`
   - `npm -v`
   - `npm install`
2) Port hygiene (prevents EADDRINUSE/ECONNREFUSED):
   - `lsof -i :5000 || true`
   - `kill -9 $(lsof -t -i :5000) 2>/dev/null || true`
   - `lsof -i :3000 || true`
   - `kill -9 $(lsof -t -i :3000) 2>/dev/null || true`
3) Baseline gates:
   - `npm run test:auth || npx vitest run tests/auth.test.mjs`
   - `npm run audit`
Record pass/fail + key excerpts.

PHASE 2 — QA GUARDIAN (Determinism + Auth/Admin)
Goal: auth tests never depend on a manually running server.
1) Inspect `package.json` scripts and `tests/auth.test.mjs`.
2) Implement the smallest deterministic harness:
   Preferred: modify `test:auth` to:
   - spawn server as child process
   - wait for `/api/health` (poll w/ timeout)
   - run vitest
   - always terminate server process
   Alternative: vitest `beforeAll/afterAll` to start/stop server.
3) Security/status rules (apply only if needed for tests):
   - missing fields → 400 (generic safe message)
   - invalid creds OR rate-limited → 401 “Invalid credentials” (uniform; no disclosure)
   - unauthenticated → 401
   - authenticated non-admin on admin routes → 403
4) Verify until green:
   - `npm run test:auth || npx vitest run tests/auth.test.mjs`

STOP condition: no ECONNREFUSED/EADDRINUSE + tests green.

PHASE 3 — UX FINISHER (Fix “washed-out” UI + mode + nav + 404)
Goal: premium healing-luxury visuals with stronger contrast and hierarchy, token-driven.

A) Correct the “washed-out / wireframe” symptoms (evidence-based)
Implement minimal changes that:
- Increase text contrast using existing tokens/semantic classes
- Reduce divider dominance: replace repeated thin lines with spacing + surface separation
- Strengthen CTA hierarchy: primary actions unmistakable
- Improve card separation: consistent radius + subtle shadow (reduced in low-stim)
No new token names; no hex.

B) Unified UI system (minimal diffs)
- Buttons: primary/secondary/ghost/disabled unified padding/radius/focus ring
- Links: teal text, underline on hover only, gold focus ring on keyboard focus
- Forms: consistent input border/hover/focus/error styles using tokens
- Typography: Playfair for headings, Inter for body; consistent line-height + spacing
- Icons: 24px baseline, outline feel, token colors only
- Motion: respects prefers-reduced-motion; low-stim reduces transitions/animations

C) Mode system (Default / Low-Stim / Reading)
Requirements:
- use `document.documentElement.dataset.mode`:
  Default: unset or ""
  Low-Stim: "low-stim"
  Reading: "reading"
- Toggle UI in TWO places:
  Header top-right
  Settings page (Display/Mode section with brief descriptions)
- Persistence via `localStorage` key `glp-mode`
- Apply pre-paint (earliest init in `client/src/main.jsx` or equivalent)
- A11y: keyboard reachable; aria-pressed/aria-current; gold focus ring via tokens

D) Navigation lockdown (zero drift)
- treat App.jsx routes as canonical
- find all internal links and align exact paths
- fix or remove broken/missing routes (do not invent routes)

E) NotFound (premium 404)
- Playfair headline + calm supportive copy
- Go Home (primary) + Go Back (secondary)
- Quick Links (4–6) ONLY routes from App.jsx
- responsive spacing

F) Verify UX gates:
- `npm run audit`
Confirm:
- NAV_LINK_AUDIT = 0 broken
- VISUAL_DOCTOR = 0 violations
- no hex outside brand-tokens.css
- fonts only Inter/Playfair

Create/update:
- docs/COLOR_SYSTEM.md (tokens list, semantic mapping, mode deltas, do/don’t)
- docs/TYPOGRAPHY_GUIDE.md (Inter/Playfair hierarchy, spacing, line-height)
