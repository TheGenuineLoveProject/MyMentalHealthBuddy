--------------------------------------------------------------------------------
2A) scan-duplicates.mjs — REQUIRED BEHAVIOR
--------------------------------------------------------------------------------
It MUST:
1) Compute fingerprints for ALL relevant files:
   - For JS/TS/TSX/JSX/JSON/MD:
     - normalized text (strip whitespace, normalize line endings)
     - sha256 hash of normalized content
   - For images/binary:
     - raw sha256
2) Detect duplicates in 4 ways:
   A) Exact duplicates (same hash)
   B) Near-duplicates (similarity score using token-based similarity)
   C) Name duplicates (same basename in multiple dirs)
   D) “Shadow copies” in backup folders (e.g. /backup, /old, /copy, /tmp, /dist, /build)
3) Detect “duplicate modules” risk:
   - multiple package.json with same "name"
   - multiple lockfiles
   - duplicate tsconfig variants that conflict
4) Output:
   - /docs/duplicates/scan-latest.md
   - /docs/duplicates/locks.md (create if missing)
5) Create a machine-readable artifact:
   - /docs/duplicates/scan-latest.json

“scan-latest.md” MUST include:
- Summary table: exact dup count, near dup count, name dup count, risky dirs
- Top 25 most dangerous duplicates (by risk score)
- Suggested action per group: KEEP, LOCK, REDIRECT-IMPORTS, QUARANTINE, DELETE (but do NOT delete automatically)

Risk scoring (must implement):
- +5 if in src/client/server
- +5 if referenced by imports
- +4 if named index.ts(x)/App.tsx/routes.ts/schema.ts
- +3 if in auth/stripe/openai/db
- +2 if in backup/copy folders
- -3 if in dist/build/node_modules
