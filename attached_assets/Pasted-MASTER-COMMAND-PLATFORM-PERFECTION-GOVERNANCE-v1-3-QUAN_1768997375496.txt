AGENT 1 — QA GUARDIAN (LOGIN + ADMIN + DETERMINISM)
Goal: auth/admin tests are deterministic; no ECONNREFUSED/EADDRINUSE; login/admin flows stable.

1) Determine expectations:
- Inspect `package.json` scripts (test:auth, dev, start, build if present)
- Read `tests/auth.test.mjs` for BASE_URL, endpoints, and security expectations
- Identify admin routes/guards (server code) and any admin test coverage

2) Fix root cause of ECONNREFUSED/EADDRINUSE:
Implement MINIMUM deterministic harness so tests never require a manual dev server:
Option A (preferred, smallest reliable): test script starts server as child process → wait for `/api/health` → run vitest → stop server (always).
Option B: vitest beforeAll/afterAll (auth tests only) to spawn/kill server.

Rules:
- Add timeouts + retries around health checks (bounded)
- Ensure server teardown executes even if tests fail
- Do not change port numbers unless required by failing gates

3) Auth security/status consistency (apply only as required by tests):
- Missing required fields → 400 with safe generic message
- Invalid credentials OR rate-limited login → 401 “Invalid credentials” (uniform; never reveal rate limiting)
- Unauthenticated access → 401
- Authenticated non-admin access to admin routes → 403
- Ensure error bodies never leak stack traces, DB messages, or internal IDs

4) Admin login readiness (do not scope creep):
- If admin login fails due to missing seeded admin: implement minimal seed mechanism that works in dev/test only:
  - Prefer an environment-guarded seed (e.g., TEST_SEED_ADMIN=1) or a deterministic test fixture
  - Do NOT add new dependencies
  - Do NOT store plaintext secrets; use existing hashing if present
- If the system already supports admin creation, align tests + docs to use it (minimal change)

5) Verify until green:
- `npm run test:auth || npx vitest run tests/auth.test.mjs`
Stop Agent 1 only when:
- tests are GREEN
- no ECONNREFUSED / EADDRINUSE appears in test runs
