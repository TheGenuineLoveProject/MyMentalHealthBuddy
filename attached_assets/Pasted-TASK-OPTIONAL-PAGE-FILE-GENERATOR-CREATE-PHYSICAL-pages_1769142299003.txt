TASK: OPTIONAL PAGE FILE GENERATOR — CREATE PHYSICAL /pages FILES FOR ALL ROUTES (NO BROKEN IMPORTS)

GOAL:
Autopilot already renders routes dynamically, but some teams want explicit file routing.
Create a generator script that creates/updates real /pages/*.jsx files for:
- every STATIC route in /content/routes.js (excluding aliases that should redirect or reuse)
- dynamic routes:
  - /blog/:slug -> /pages/blog/[slug].jsx
  - /community/discussion/:id -> /pages/community/discussion/[id].jsx
- NotFound:
  - /pages/404.jsx
Do NOT generate duplicates for aliases; make aliases redirect to canonical using Next.js redirect (or render canonical).

HARD RULES:
- Next.js pages router
- Must compile
- No TODOs
- Do not break existing /pages/_app.jsx, /pages/_document.jsx, or /pages/_autopilot.jsx
- Generator must be safe to rerun (idempotent)

IMPLEMENT:
1) Add script file:
- /scripts/generate-pages.mjs   (Node ESM)
It must:
- import { routes } from "../content/routes.js" (or require if CJS; match project style)
- determine which routes are:
  a) static (no ":" and not "/*")
  b) dynamic patterns (":slug", ":id")
  c) aliases (aliasOf exists)
- create directories under /pages to match route path
- write .jsx files for each static route that render via PageTemplate:
  Example file contents:
    import PageTemplate from "@/components/PageTemplate";
    import { getRouteConfig } from "@/content/routes";
    export default function Page() { return <PageTemplate config={getRouteConfig("/path")} />; }
- for aliases:
  choose ONE approach (pick and implement fully):
  OPTION A (preferred): render canonical config:
    getRouteConfig("/canonical")
  OPTION B: use next/router redirect (client-side) with minimal flash
  OPTION C: use getServerSideProps redirect (server-side)
Pick ONE and apply consistently.

2) Dynamic pages:
- /pages/blog/[slug].jsx:
  - getStaticPaths returns empty paths with fallback: "blocking" OR minimal stub
  - getStaticProps pulls config via getRouteConfig(`/blog/${params.slug}`)
  - render PageTemplate
- /pages/community/discussion/[id].jsx similarly

3) 404:
- /pages/404.jsx renders PageTemplate with getRouteConfig("/*") or a dedicated notFound config

4) Add npm script:
- in package.json add:
  "gen:pages": "node scripts/generate-pages.mjs"
Do not break JSON formatting.

5) Output requirements:
- Provide FULL file contents for:
  - /scripts/generate-pages.mjs
  - any newly created/modified page templates (only if necessary)
  - package.json (only if modified; full file)
- Provide "GENERATOR REPORT":
  - number of pages created
  - list of generated file paths
  - how aliases were handled
  - confirm dynamic pages created

DONE GATE:
✅ npm run dev works
✅ npm run gen:pages runs without errors
✅ generated pages compile and render correct configs
✅ no duplicates for aliases
FAIL THE RUN if any DONE GATE item is not satisfied.
