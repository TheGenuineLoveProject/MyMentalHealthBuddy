PHASE 0 — BASELINE VERIFY (NO CHANGES)
Run and print:
- npm run verify
If FAIL, STOP and fix before continuing.

PHASE 1 — UTM BUILDER (CANONICAL)
Create:
- shared/utm.mjs (or shared/utm.ts if shared uses TS) with:
  buildUtmUrl(baseUrlOrPath, { source, medium, campaign, content })
Rules:
- Must support internal paths ("/blog") and absolute URLs.
- Must never include PII (no email, no userId).
- Provide defaults:
  source = "glp"
  medium = "owned"
  campaign = "publishing"
- Add small unit test or script-based check:
  scripts/test-utm.mjs prints sample URLs

PHASE 2 — ANONYMOUS CLICK EVENT (SERVER)
Create route:
- server/routes/events.mjs
Mount at:
- POST /api/events/click

Payload:
{ event: "click", target: "<url-or-path>", context: { surface:"blog|newsletter|social", slug?, draft_id? }, ts }

Requirements:
- Rate limit
- Validate payload (zod or existing validator)
- Store in DB table analytics_events if it exists; else fallback:
  data/analytics_events.jsonl append-only

Return:
{ ok:true }

PHASE 3 — CLIENT TRACKING (MINIMAL + ETHICAL)
Add a tiny helper:
- client/src/utils/trackClick.js (or .ts)
that calls POST /api/events/click
Use it ONLY for:
- Newsletter CTA buttons
- Blog newsletter CTA block
- Pricing CTA (optional)
No global surveillance. No keystroke tracking. No session replay.

PHASE 4 — ADMIN SIGNALS DASHBOARD
Create or extend:
- /admin/publishing/signals

Must show:
- Top clicked targets (last 7 days)
- Top surfaces (blog/newsletter/social)
- Top campaigns/content tags (if present)
- Simple recommendation box:
  - “Double down on pillar X”
  - “This CTA underperformed”
No auto-changes. Human decides.
