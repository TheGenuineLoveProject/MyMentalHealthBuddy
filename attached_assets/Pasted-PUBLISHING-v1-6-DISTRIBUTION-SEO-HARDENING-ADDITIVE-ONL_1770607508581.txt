PUBLISHING v1.6 — DISTRIBUTION & SEO HARDENING (ADDITIVE ONLY)

HARD RULES:
- ADDITIVE ONLY. No rename/delete/move/refactor existing files.
- Manual-first: no auto-posting; newsletter send must be human-approved.
- Ethical language only: no urgency/shame/dependency; no medical claims.
- Must end with PASS/FAIL outputs + VERIFIED: PASS/FAIL via npm run verify.

GOAL:
Make Blog + Newsletter + Publishing robust, discoverable, and distribution-ready:
1) Add canonical RSS feed
2) Add sitemap.xml generator
3) Add blog meta/OG consistency gates
4) Add newsletter sequences (welcome + weekly digest drafts)
5) Add “post distribution pack” generator (copy-ready captions)
6) Extend audits + verify

PHASE 0 — BASELINE VERIFY (NO CHANGES)
Run:
- npm run verify
If FAIL, STOP.

PHASE 1 — RSS FEED (CANONICAL)
Create:
- server/routes/rss.mjs
Expose:
- GET /rss.xml
Requirements:
- Uses published blog posts only (drafts excluded)
- If DB blog_posts exists, query it; otherwise read from content/blog/*.md or content/blog/posts.json (detect existing source)
- Output valid RSS 2.0 XML
Print:
RSS_READY: PASS

PHASE 2 — SITEMAP GENERATOR
Create:
- scripts/generate-sitemap.mjs
Output:
- public/sitemap.xml
Include:
- core routes: /, /blog, /newsletter, /pricing, /crisis, /contact, /faq
- all published blog post URLs
- update timestamp
Add npm script:
- "sitemap": "node scripts/generate-sitemap.mjs"
Print:
SITEMAP_READY: PASS

PHASE 3 — META/OG CONSISTENCY GATES
Create:
- scripts/audit-seo.mjs
Check:
- each published blog post has:
  title, description/excerpt, canonical URL, og:title, og:description
- ensure no empty meta tags
Output:
SEO_AUDIT: PASS/FAIL (list failures)

PHASE 4 — NEWSLETTER SEQUENCES (DRAFTS ONLY)
Create canonical drafts (no auto-send):
- content/newsletter/sequences/welcome_3part.json
- content/newsletter/sequences/weekly_digest_template.json

Each email must include:
- subject
- preview_text
- body
- gentle CTA (blog/newsletter/pricing optional)
- safety note when relevant
- disclaimer footer (non-clinical)
Status fields:
draft / approved / sent

PHASE 5 — DISTRIBUTION PACK GENERATOR (COPY-READY)
Create:
- scripts/generate-distribution-pack.mjs

Input:
- a blog post slug or draft_id
Output (prints + writes file):
- exports/distribution/<slug>/
  - instagram.txt
  - tiktok.txt
  - x.txt
  - youtube_shorts.txt
  - newsletter_snippet.txt

Rules:
- CTA links must use UTM builder if it exists (shared/utm.*)
- Must include “soft CTA” and avoid urgency language
Output:
DISTRIBUTION_PACK: PASS

PHASE 6 — ADMIN “PUBLISHING CHECKLIST” PANEL
Add page:
- /admin/publishing/checklist
Show:
- Latest 10 blog posts (published/draft)
- Checklist columns:
  links_ok, tone_ok, seo_ok, utm_ok, distro_pack_ready
Buttons:
- “Generate distro pack”
- “Run audits”
Manual-only.

PHASE 7 — UPGRADE AUDIT PIPELINE + VERIFY
Update:
- scripts/audit-publishing.mjs to also run:
  - node scripts/audit-seo.mjs
  - npm run sitemap
  - (optional) node scripts/generate-distribution-pack.mjs --dry-run on latest post
Update:
- scripts/verify.mjs includes publishing audits + build + health check

PHASE 8 — REQUIRED FINAL COMMANDS (RUN + PASTE OUTPUT)
- npm run sitemap
- node scripts/audit-seo.mjs
- curl http://localhost:5000/rss.xml (or project port) and paste first 10 lines
- node scripts/generate-distribution-pack.mjs --dry-run
- npm run verify

FINAL REPORT REQUIRED:
A) Files created/changed (full paths)
B) Evidence outputs showing PASS lines + VERIFIED line
C) What is now discoverable/distributable (3 bullets max)
D) Next 3 steps only
