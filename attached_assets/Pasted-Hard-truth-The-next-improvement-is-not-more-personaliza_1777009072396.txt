TARGET FILES:

GOAL:
Make Buddy emotionally safe, accessible, keyboard-safe, reduced-motion-safe, and screen-reader friendly.
PHASE 1 — REDUCED MOTION
Ensure BuddyAvatar respects prefers-reduced-motion:
- no pulsing animation
- no floating animation
- no sparkles
- static visual state only
- preserve color/state changes
PHASE 2 — ARIA LABEL
BuddyAvatar should expose a meaningful label:
aria-label="Buddy avatar showing [state] support mode"
Example:
calm → "Buddy avatar showing calm support mode"
anxious → "Buddy avatar showing anxious breathing support mode"
crisis → "Buddy avatar showing crisis-safe support mode"
PHASE 3 — DECORATIVE PARTS
Internal visual-only pieces:
- eyes
- heart
- body highlights
- base
should use aria-hidden="true"
PHASE 4 — CONTRAST + SAFETY
Ensure:
- no flashing
- no rapid color changes
- crisis mode uses steady calm color, not alarm red
- heart/eyes remain visible against face/body
- copy near Buddy remains readable
PHASE 5 — KEYBOARD SAFETY
If Buddy has any interactive container:
- remove it unless necessary
- if clickable, provide button role + keyboard handling
- otherwise make Buddy non-interactive visual feedback only
PHASE 6 — SCREEN READER COPY
Near Buddy on /start, include subtle visible copy and/or sr-only copy:
"Buddy is a visual companion. Your support tools still work without animation."
PHASE 7 — TELEMETRY
Add only if existing tracking helper is already available:
buddy_accessibility_ready
Do not log message text.
PHASE 8 — VERIFY
Run:
npm run dev
Manual checks:
1. /start loads
2. Buddy appears
3. no console errors
4. keyboard tab order is not disrupted
5. reduced motion mode does not animate
6. crisis state is steady, calm, non-flashing
7. /api/ai/chat still works
8. /api/buddy still works
Return:
- files changed
- accessibility improvements
- verification results
- skipped changes
