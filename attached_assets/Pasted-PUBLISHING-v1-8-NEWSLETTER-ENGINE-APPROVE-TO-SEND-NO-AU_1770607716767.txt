- Ethical language: no urgency/shame/dependency; no medical claims.
- Must end with PASS/FAIL evidence + VERIFIED: PASS/FAIL via npm run verify.

GOAL:
1) Store newsletter drafts + status in DB (or file-backed fallback)
2) Create an admin newsletter dashboard to create/approve/send
3) Use Resend integration to send safely

5) Add verification scripts and safety gates

PHASE 1 — DATA MODEL (DB FIRST, FALLBACK SECOND)
Check if tables exist:
- newsletter_campaigns
- newsletter_subscribers
If missing, ADD new tables via Drizzle (shared/schema.mjs):
- newsletter_campaigns:
  id, subject, preview_text, html_body, text_body, status(draft|approved|sent),
  created_at, approved_at, sent_at, created_by, approved_by, audience(all|segment)
- newsletter_subscribers:
  id, email, status(active|unsubscribed), source, created_at, unsubscribed_at

If DB changes are heavy, implement file-backed MVP:
- content/newsletter/campaigns/*.json with same fields
But prefer DB since you already run Neon + Drizzle.

PHASE 2 — NEWSLETTER API (MANUAL-FIRST)
Create/confirm server routes:
- server/routes/newsletter-admin.mjs (admin-only)
Endpoints:
- GET /api/admin/newsletter/campaigns
- POST /api/admin/newsletter/campaigns (create draft)
- POST /api/admin/newsletter/campaigns/:id/approve (status -> approved)
- POST /api/admin/newsletter/campaigns/:id/send (status must be approved)
- POST /api/newsletter/subscribe (public)
- POST /api/newsletter/unsubscribe (public)

Rules:
- /send must enforce: status === approved, subject present, body present
- Add rate limiting to subscribe/unsubscribe
- Admin routes require adminAuth middleware

PHASE 3 — RESEND SEND IMPLEMENTATION (SAFE)
Use existing Resend integration (server/routes/email.mjs or server/services).
Add a single send function:
- server/services/newsletterSend.mjs
Features:
- Batch sending with small batch size (e.g., 50)
- Logs send counts (no PII logs)
- Handles failures gracefully and returns summary
- Supports “dry run” mode: sends only to ADMIN_EMAIL if set

ENV:
- RESEND_API_KEY must exist OR skip sending with clear error
- ADMIN_EMAIL optional for dry runs
