STEP 1 — FIX “VIEW DISCUSSION” BUTTONS (ROOT-CAUSED)
For each “View Discussion” control:
A) Ensure it is a real navigation element:
   - If Next.js: use <Link href="..."> with an <a> or button-styled link.
   - If React Router: use <Link to="...">.
   - If plain button: add onClick={() => navigate(...)} (and ensure navigate is defined).
B) Make the click target large:
   - Use padding and display inline-flex; min-height 44px; min-width 44px.
C) Prevent click blocking:
   - Inspect parent containers for absolute overlays, full-width elements, pseudo-elements, or pointer-events: none.
   - Fix with: position: relative; z-index; pointer-events: auto on the button; pointer-events: none only on decorative layers.
D) Confirm route exists:
   - If route is missing but intended, add a minimal UI page/route shell (UI-only) that displays “Discussion” and placeholders.
