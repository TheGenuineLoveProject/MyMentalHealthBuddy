PHASE 1 — EDITORIAL CALENDAR (CANONICAL)
Create:
- content/publishing/calendar/editorial_calendar.json

Rules:
- 14 entries (next 14 days)
- fields:
  date, channel (newsletter/blog/social), pillar, title, slug(optional), draft_id(optional), status(planned/draft/approved/published/sent), notes
- rotate pillars across: Orientation, Reflection, Emotional Literacy, Practice, Community, Safety, Platform Feature, Blog highlight
- include at least 4 newsletter sends, 4 blog posts, 6 social posts (copy-first)

PHASE 2 — “TODAY’S PUBLISH” ADMIN PANEL (MANUAL-FIRST)
Create or extend admin:
- /admin/publishing/today

Must show:
- Today’s calendar item(s)
- If newsletter item: show draft_id + copy buttons (subject/body/cta)
- If blog item: show slug + draft template
- If social item: show captions + copy buttons
Must allow status updates (planned→approved→sent/published) stored in:
- data/publishing_status.json

PHASE 3 — LINK INTEGRITY SCAN (NO BROKEN PUBLIC SURFACE)
Create script:
- scripts/audit-links.mjs

It must:
- Scan blog posts + newsletter drafts + templates for URLs that start with "/"
- Confirm route targets exist in routing registry:
  - at minimum these must exist: /blog, /newsletter, /pricing, /crisis, /contact
- Print:
  LINKS_AUDIT: PASS
  or
  LINKS_AUDIT: FAIL (missing targets listed)

PHASE 4 — QUALITY GATES (TONE + SAFETY)
Create script:
- scripts/audit-tone.mjs

It must scan newsletter drafts + templates + blog posts for forbidden patterns:
- “cure”, “treat”, “diagnose”, “guaranteed results”
- urgency: “today only”, “limited time”, “don’t miss”
- shame: “what’s wrong with you”, “you should be ashamed”
- dependency: “you need us”, “only we can”
Output:
  TONE_AUDIT: PASS
or
  TONE_AUDIT: FAIL (examples)

Update scripts/verify.mjs to run:

Then final line:
  VERIFIED: PASS
or
  VERIFIED: FAIL
