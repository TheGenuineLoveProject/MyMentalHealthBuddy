TASK: GENERATOR SCRIPT (optional physical pages + modes + overwrite safety + manifests + hashes)

GOAL:
Create a deterministic generator that can write physical route files under:
client/src/pages/generated/
even though autopilot already works.

MUST SUPPORT MODES:
1) --mode=all
2) --mode=landing   // only Landing routes (7)
3) --mode=category --category="Wellness & Healing Tools"
4) --mode=range --from=A --to=C  // letter range mapped via /content/categoryOrder.js

SOURCE OF TRUTH:
- Add /content/categoryOrder.js exporting:
  - export const CATEGORY_ORDER = [
      { letter:"A", category:"Landing" }, { letter:"B", category:"Authentication" }, ...
    ]
This must be used by generator AND design playbook.

OVERWRITE SAFETY (STRICT):
- Generator MUST refuse to overwrite any existing page file unless it contains a top-line header:
  // @generated
- If file exists without that header, skip and report it.

OUTPUT ARTIFACTS (REQUIRED):
- reports/routes.generated.json (manifest of generated files + route metadata)
- reports/routes.config.snapshot.json (snapshot of route configs used)
- reports/routes.generated.sha256 (stable hash of the manifest)
- reports/routes.config.sha256 (stable hash of config snapshot)
- reports/routes.config.hashes.json (per-route hash table so CI can pinpoint changes)

ALSO REQUIRED:
- A stable hash of the route configs themselves EVEN IF manifest wasn’t regenerated.
  - Write reports/routes.config.sha256 derived from normalized config snapshot.
- Print totals: files written, files skipped, errors.

IMPLEMENT:
- scripts/gen-pages.mjs (reads /content/routes.js and writes files)
- Add npm scripts:
  - gen:pages:all
  - gen:pages:landing
  - gen:pages:category -- --category="..."
  - gen:pages:range -- --from=A --to=C

PAGE OUTPUT PATTERN (must match your existing output):
- Write files into client/src/pages/generated/
- Use deterministic route->file mapping rules (see mapping task below).
- Each generated file must include // @generated and render the config-driven PageTemplate for that route.

OUTPUT FORMAT:
1) IMPLEMENTATION CHECKLIST (before)
2) Full code for:
   - /content/categoryOrder.js
   - /scripts/gen-pages.mjs
   - package.json scripts changes
3) Example run commands for each mode
4) COMPLETED CHECKLIST (after)
DONE GATE:
- Running gen:pages:landing then gen:pages:all produces stable artifacts
- No overwrites occur without // @generated
