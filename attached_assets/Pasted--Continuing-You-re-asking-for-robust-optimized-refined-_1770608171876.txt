✅ EXACT REPLIT AI PROMPT — PUBLISHING v2.1 “NARRATIVE AMPLIFICATION PACK” (ETHICAL + ON-BRAND)

PUBLISHING v2.1 — NARRATIVE AMPLIFICATION PACK (25 EVERGREEN POSTS + WEEKLY NEWSLETTER DRAFTS)

HARD RULES:
- ADDITIVE ONLY. Do not rename/delete/move/refactor existing files.
- No background autonomy. No auto-posting. Human approval required for anything marked “approved/published”.
- Must be compatible with the canonical publishing registry from v1.9:
  content/publishing/publishingRegistry.json
- Must respect pillars taxonomy from v1.9:
  content/publishing/pillars.json
- Must pass duplicate prevention:
  node scripts/prevent-duplicates.mjs
- Must end with PASS/FAIL evidence + VERIFIED via npm run verify.

GOAL:
1) Populate a high-quality, evergreen publishing library:
   - 25 social drafts (daily posts)
   - 12 blog drafts (1 per pillar, evergreen)
   - 4 newsletter drafts (weekly)
2) Ensure every item is safely written:
   - no medical claims
   - no urgency/shame
   - no dependency framing
   - includes soft CTA to /newsletter, /blog, /tools, or /crisis (when relevant)
3) Add Admin tooling to “Approve for Today” and “Copy captions” quickly.

PHASE 0 — BASELINE VERIFY (NO CHANGES)
Run:
- npm run verify
If FAIL, STOP.

PHASE 1 — CREATE CANONICAL CONTENT SEEDS (ONE SOURCE OF TRUTH)
Create:
- content/publishing/drafts/socialPack_v2_1.json  (25 entries)
- content/publishing/drafts/blogPack_v2_1.json    (12 entries)
- content/publishing/drafts/newsletterPack_v2_1.json (4 entries)

Each entry MUST include:
- id (glp-post-YYYYMMDD-xxx or glp-seed-xxx)
- type: social | blog | newsletter
- pillar: one of pillars.json keys
- title (for blog/newsletter)
- captions:
   - instagram
   - x
   - tiktok
   - youtubeShorts
- body (blog/newsletter only) as “draftMarkdown”
- primaryCta (one of /newsletter /blog /tools /pricing /crisis)
- safetyNote (1 sentence)
- status: draft
- createdAt ISO

Quality constraints:
- Tone: warm, grounded, non-clinical, invitational.
- Add a small “gentle CTA” line at end.
- For any content referencing crisis/emergency: include /crisis CTA and “If you’re in immediate danger, call local emergency services.”

PHASE 2 — REGISTER DRAFTS INTO publishingRegistry.json (WITHOUT DUPLICATION)
Implement a safe append step:
- Create script: scripts/import-publishing-drafts.mjs
Behavior:
- Reads the three pack files
- Validates schema
- Inserts into publishingRegistry.json ONLY if id does not already exist
- Never overwrites existing items
Output:
IMPORT_DRAFTS: PASS (inserted N, skipped M)

PHASE 3 — ADMIN “TODAY’S PICK” WORKFLOW (MANUAL-FIRST)
Create:
- client/src/pages/admin/AdminPublishingToday.jsx
Route: /admin/publishing/today
Features:
- Shows “Drafts ready”
- One-click set “Today’s featured item”
- Copy buttons per platform caption (IG/X/TikTok/Shorts)
- “Mark as posted” (sets status to posted + postedAt)
Storage:
- Use existing admin settings storage if present; otherwise create minimal storage:
  - server/routes/admin-publishing.mjs additions OR a small DB table: publishing_featured (date, glp_id)
Guard:
- adminAuth required

PHASE 4 — BLOG + NEWSLETTER WIRES (SAFE STUBS ONLY)
If blog system exists:
- Ensure there is a canonical blog route that can render draftMarkdown by slug/id.
If not:
- Create minimal read-only rendering route:
  - client/src/pages/BlogDraftViewer.jsx
  - route: /blog/draft/:id (admin only)
This is for review ONLY, not public publishing.

PHASE 5 — VALIDATION + SAFETY CHECKS
Create:
- scripts/validate-draft-packs.mjs
Checks:
- each draft conforms to schema
- pillar keys valid
- CTA path allowed
- no forbidden phrases list (medical claims, “cure”, “guarantee”, “diagnose”, “treat”, etc.)
Output:
DRAFT_PACKS_VALIDATE: PASS/FAIL

PHASE 6 — VERIFY
Run required:
- node scripts/validate-draft-packs.mjs
- node scripts/import-publishing-drafts.mjs
- node scripts/prevent-duplicates.mjs
- npm run verify

FINAL REPORT REQUIRED:
A) Files created/changed (full paths)
B) PASS/FAIL logs + VERIFIED line
C) Screenshot proof of /admin/publishing/today
D) Next 2 steps only (v2.2 + v2.3)
