TARGET FILE:

client/src/components/avatar/BuddyAvatar.tsx only if a small visual share badge is needed.
GOAL:
After a successful non-crisis tool response, show a gentle share prompt:
"Buddy helped me reset for one minute."
The share must help growth without exposing user emotion or private text.
PHASE 1 — SHARE ELIGIBILITY
Only show share prompt if:
- response is successful
- response has a tool
- crisis is false
- user has completed or received a tool response
Never show share prompt if:
- response contains crisis state
- response is error
- no tool exists
PHASE 2 — SHARE COPY
Use privacy-safe copy only:
Primary:
"Buddy helped me reset for one minute. Try MyMentalHealthBuddy: /start"
Alternate:
"I took a 60-second reset with Buddy. Try it here: /start"
Do not include:
- the user's message
- the AI reply
- inferred emotional state
- profile data
- streak data
- paywall data
PHASE 3 — SHARE BUTTON
Add a share card below the completed tool area:
Title:
"Share a reset"
Body:
"If this helped, someone else may need a gentle reset too."
Button:
"Share Buddy"
Use:
- navigator.share if available
- navigator.clipboard.writeText fallback if not
PHASE 4 — BUDDY VISUAL SUPPORT
Optional small copy near Buddy:
"Buddy can help someone else reset too."
Do not animate aggressively.
PHASE 5 — TELEMETRY
Track additive-only:
buddy_share_shown
buddy_share_clicked
Metadata:
{
  toolId,
  buddyState
}
Do not log message text.
Do not log AI reply.
Do not log crisis content.

npm run dev

- tool response appears
- share prompt appears

- share button works

3. Crisis input

- share prompt does not appear
4. Error response
