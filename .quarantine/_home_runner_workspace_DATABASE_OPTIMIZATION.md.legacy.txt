# Database Optimization Guide - MyMentalHealthBuddy Platform

## Overview
This document outlines recommended database optimizations to improve query performance and scalability for the MyMentalHealthBuddy platform.

## Current Status
✅ Database schema is well-designed with proper foreign key relationships  
✅ Primary keys use UUIDs for distributed system compatibility  
⚠️ Missing explicit indexes on frequently queried columns  
⚠️ No composite indexes for common query patterns  

## Recommended Indexes

### High-Priority Indexes (Immediate Impact)

#### User-Related Queries
```sql
-- Frequently queried by subscription tier and status
CREATE INDEX idx_users_subscription_tier ON users(subscription_tier);
CREATE INDEX idx_users_subscription_status ON users(subscription_status);
CREATE INDEX idx_users_stripe_customer_id ON users(stripe_customer_id);
```

#### Healing Messages (Chat History)
```sql
-- Session-based chat retrieval
CREATE INDEX idx_healing_messages_session_id ON healing_messages(session_id);
CREATE INDEX idx_healing_messages_user_id ON healing_messages(user_id);
-- Time-based queries
CREATE INDEX idx_healing_messages_timestamp ON healing_messages(timestamp DESC);
```

#### Journals & Mood Entries
```sql
-- User journal retrieval
CREATE INDEX idx_journals_user_id_created_at ON journals(user_id, created_at DESC);
-- Mood analytics
CREATE INDEX idx_mood_entries_user_id_created_at ON mood_entries(user_id, created_at DESC);
```

#### Billing Transactions
```sql
-- Transaction history
CREATE INDEX idx_billing_transactions_user_id_created_at ON billing_transactions(user_id, created_at DESC);
CREATE INDEX idx_billing_transactions_status ON billing_transactions(status);
```

#### Crisis Resources
```sql
-- Country-based lookup
CREATE INDEX idx_crisis_resources_country_active ON crisis_resources(country, is_active);
```

### Medium-Priority Indexes

#### Social Media Integration
```sql
CREATE INDEX idx_social_accounts_user_id_platform ON social_accounts(user_id, platform);
CREATE INDEX idx_social_posts_history_user_id_published_at ON social_posts_history(user_id, published_at DESC);
CREATE INDEX idx_social_posts_history_account_id ON social_posts_history(account_id);
```

#### Content Management
```sql
CREATE INDEX idx_content_posts_user_id_status ON content_posts(user_id, status);
CREATE INDEX idx_scheduled_posts_user_id_scheduled_for ON scheduled_posts(user_id, scheduled_for);
```

#### AI & Media
```sql
CREATE INDEX idx_ai_prompts_user_id ON ai_prompts(user_id);
CREATE INDEX idx_ai_runs_user_id_created_at ON ai_runs(user_id, created_at DESC);
CREATE INDEX idx_media_assets_user_id_created_at ON media_assets(user_id, created_at DESC);
```

#### Analytics
```sql
CREATE INDEX idx_user_activity_user_id_created_at ON user_activity(user_id, created_at DESC);
CREATE INDEX idx_analytics_snapshots_user_id_snapshot_date ON analytics_snapshots(user_id, snapshot_date DESC);
```

### Low-Priority Indexes (Future Optimization)

#### Automation
```sql
CREATE INDEX idx_automation_rules_user_id_active ON automation_rules(user_id, is_active);
CREATE INDEX idx_bulk_operations_user_id_status ON bulk_operations(user_id, status);
```

#### Audit & Security
```sql
CREATE INDEX idx_audit_logs_user_id_created_at ON audit_logs(user_id, created_at DESC);
CREATE INDEX idx_audit_logs_resource_type_resource_id ON audit_logs(resource_type, resource_id);
```

## Query Optimization Best Practices

### Current Implementations
✅ Query caching with TTL (apps/server/src/lib/queryOptimizer.ts)  
✅ Pagination helpers to limit result sets  
✅ Connection pooling optimized for Replit Autoscale  

### Recommended Practices
1. **Use prepared statements** - Already implemented via Drizzle ORM
2. **Limit result sets** - Always use pagination for list queries
3. **Cache frequently accessed data** - Use the QueryOptimizer for static/slow data
4. **Avoid N+1 queries** - Use joins or batch loading
5. **Monitor slow queries** - Log queries >500ms for optimization

## Drizzle ORM Index Implementation

To implement these indexes in Drizzle ORM, add to the table definitions in `apps/shared/db-schema.ts`:

```typescript
import { pgTable, varchar, index } from "drizzle-orm/pg-core";

export const journals = pgTable("journals", {
  // ... existing columns
}, (table) => ({
  // Add indexes here
  userIdCreatedAtIdx: index("idx_journals_user_id_created_at")
    .on(table.userId, table.createdAt),
}));
```

Then run:
```bash
npm run db:push
```
