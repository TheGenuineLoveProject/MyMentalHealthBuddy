## ✅ CRITICAL FIXES COMPLETED

### 1. **Frame Blocking Issue - RESOLVED** 🎯

**Problem:** Webpages were not visible in Replit's preview pane due to restrictive frame policies blocking the iframe.

**Root Cause:**
- `X-Frame-Options: DENY` blocked ALL iframe embedding
- `frame-ancestors 'none'` in CSP blocked external origins
- Replit's webview requires `https://replit.com` and `https://*.replit.dev` origins

**Solution Applied:**

**Security Headers (`apps/server/src/lib/securityHeaders.ts`):**
```typescript
// CSP frame-ancestors - Allow Replit origins in development
frameAncestors: isProduction 
  ? ["'self'"] 
  : ["'self'", "https://replit.com", "https://*.replit.com", "https://*.replit.dev"]
```

**Helmet Configuration (`apps/server/src/index.ts`):**
```typescript
app.use(helmet({
  contentSecurityPolicy: false,
  crossOriginEmbedderPolicy: false,
  frameguard: isDev ? false : { action: 'sameorigin' } // Disable in dev for Replit webview
}));
```

**Results:**
- ✅ Development: CSP allows Replit origins, no X-Frame-Options header
- ✅ Production: Strict security with `frame-ancestors 'self'` and `X-Frame-Options: SAMEORIGIN`
- ✅ **Webpages now visible in Replit preview pane!**

**Verified via curl:**
```bash
# Development (current)
frame-ancestors 'self' https://replit.com https://*.replit.com https://*.replit.dev
X-Frame-Options: (absent) ✓

# Production (when deployed)
frame-ancestors 'self'
X-Frame-Options: SAMEORIGIN
```

### 2. **Database Connection Pool - AUTOSCALE-SAFE** 🎯

**Problem:** Database pool configuration was unsafe for Autoscale deployment with multiple instances.

**Previous Configuration (UNSAFE):**
```typescript
max: 20,  // ❌ Too high for shared connection pool
min: 5,   // ❌ Each instance pre-warms 5 connections
allowExitOnIdle: false  // ❌ Prevents proper cleanup
```

**Issue:** With 4 Autoscale instances, `min:5` pre-warms 20 connections (5 × 4), exhausting the database connection limit before handling any traffic.

**Solution Applied (`apps/server/src/index.ts`):**
```typescript
const pgPool = new pg.Pool({
  connectionString: env.DATABASE_URL,
  max: 10,                        // Conservative max for multi-instance
  min: 0,                         // CRITICAL: No pre-warming per instance
  idleTimeoutMillis: 30000,       // Close idle connections after 30s
  connectionTimeoutMillis: 5000,  // 5s timeout for acquiring connection
  allowExitOnIdle: true,          // Allow pool cleanup (important for Autoscale)
  statement_timeout: 30000,       // SQL statement timeout (30s)
  query_timeout: 30000,           // Query timeout (30s)
});
```

**Benefits:**
- ✅ No pre-warming: Connections created on-demand only
- ✅ Conservative max (10): Safe for multiple instances
- ✅ Proper cleanup: `allowExitOnIdle: true` prevents connection leaks
- ✅ Timeouts configured: Prevents hanging queries

**Architect Verification:**
> "PostgreSQL pool settings have been adjusted to Autoscale-safe values (`max:10`, `min:0`, `allowExitOnIdle:true`) and otherwise look correct, supporting production readiness objectives."
