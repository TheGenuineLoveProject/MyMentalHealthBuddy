## 📦 How It Works

### Initialization Flow
1. App starts → StripeProvider loads
2. Public key validated → Stripe.js loaded
3. Context provides Stripe instance to entire app
4. Components can use `useStripe()` hook

### Checkout Flow
1. User clicks "Upgrade to Premium"
2. `useStripeCheckout` hook called
3. POST to `/api/stripe/create-subscription-checkout`
4. Backend creates Stripe checkout session
5. User redirected to Stripe's hosted checkout page
6. After payment → redirected back to success URL

### Status Monitoring
```typescript
const { stripe, isLoading, error, isTestMode } = useStripe();

if (isLoading) // Show loading state
if (error) // Show error message
if (stripe) // Stripe ready - enable payment features
```

## 🎯 Usage Examples

### Creating a Checkout Session
```typescript
import { useStripeCheckout } from "@/hooks/useStripeCheckout";

function UpgradeButton() {
  const { createSubscriptionCheckout, isLoading, error } = useStripeCheckout();

  return (
    <button 
      onClick={() => createSubscriptionCheckout({ tier: "premium" })}
      disabled={isLoading}
    >
      {isLoading ? "Processing..." : "Upgrade to Premium"}
    </button>
  );
}
```

### Using Stripe Directly
```typescript
import { useStripe } from "@/contexts/StripeContext";

function CustomPaymentForm() {
  const { stripe, isLoading } = useStripe();

  if (isLoading) return <div>Loading payment system...</div>;
  if (!stripe) return <div>Payment system unavailable</div>;

  // Use stripe instance directly
  return <div>{/* Custom payment form */}</div>;
}
```

## 🎓 Developer Notes

### Key Design Decisions
1. **Hosted Checkout**: Using Stripe's hosted checkout page (most secure, PCI-compliant)
2. **Context Pattern**: Global Stripe state via React Context (single initialization)
3. **Custom Hooks**: Encapsulated logic for reusability (useStripeCheckout)
4. **Environment Validation**: Fail-fast on misconfiguration (prevents runtime errors)
5. **TypeScript First**: Full type safety across all Stripe operations

### Performance Optimizations
- Stripe.js loaded once and cached
- Lazy loading of Stripe Elements
- No redundant initializations
- Memoized Stripe instance

### Error Handling Strategy
- Graceful degradation on initialization failure
- User-friendly error messages
- Console logging for debugging
- Retry logic not implemented (single-attempt load)
