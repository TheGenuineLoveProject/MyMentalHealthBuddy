# Zero Drift Operating Rules

## Prime Rule
Finish the current verified system before adding new systems.

## Rules
- shell commands first
- one phase at a time
- one file group at a time
- verify before expanding
- no runtime changes unless explicitly approved
- no auth, billing, crisis, healing, journal, provider, admin, chat changes without separate phase
- no hype claims without verified proof
- no manipulation, shame, fear, or dependency loops

## Required Gates
- npm run verify:all
- npm run build
- git status --short
