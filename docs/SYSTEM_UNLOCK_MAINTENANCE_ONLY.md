# System Unlock — Maintenance Only

STATUS: UNLOCKED (MAINTENANCE MODE)

---

## PURPOSE

Allow system stability work
without feature expansion or evolution.

---

## ALLOWED ACTIONS

✔ Bug fixes  
✔ Dependency updates  
✔ Security patches  
✔ Typo / copy fixes  
✔ Broken link fixes  

---

## DISALLOWED ACTIONS

❌ New features  
❌ UX redesigns  
❌ Automation changes  
❌ Content expansion  
❌ Roadmap changes  

---

## CHANGE CONTROL

Every maintenance action must:
• Be reversible
• Be documented
• Not alter user experience meaningfully

---

## EXIT CONDITION

“Roger v3.2, unlock system and proceed with [specific task].”

---

## FINAL RULE

Maintenance protects trust.
It does not evolve the product.
