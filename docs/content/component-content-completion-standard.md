# Component + Content Completion Standard

Every user-facing page should answer:
1. What is this?
2. Who is it for?
3. When should someone use it?
4. Why does it help?
5. What should the user do next?
6. What is the safety boundary?
7. Where can a user get crisis support if needed?

Every interactive component should have:
- visible label
- accessible label where needed
- deterministic route target or action
- no empty buttons
- no invisible text
- no black-on-sage contrast failure
- no dead href
- no duplicate ownership
- no unsupported clinical claim

Every wellness page should include:
- educational boundary
- not therapy / not diagnosis / not medical care language where appropriate
- crisis route visibility for sensitive topics
- clear next step
- content coherence with platform mission

Every monetization page should include:
- clear offer
- no emotional vulnerability exploitation
- no healing-data targeting
- cancellation clarity
- ethical subscription language
