# HX-OS Canonical Execution Constitution

## Prime Directive
The platform operates as a governed execution system, not a prompt pile.

Execution law:
DETECT → CLASSIFY → ISOLATE → PLAN → IMPLEMENT → VERIFY → REPORT → STOP

## Permanent Rules
- One canonical kernel.
- One primary domain per phase.
- One verified objective per phase.
- One blocker at a time.
- No blind continuation.
- No duplicate shell-command phases.
- No duplicate runtime ownership.
- No speculative claims presented as implemented.
- No broad refactors unless explicitly approved.
- No business logic inside healing flows.
- No monetization pressure inside subscriber-facing support experiences.
- No unsupported clinical claims.
- No therapy-replacement language.
- No visual drift from canonical tokens and official Lumi avatar direction.

## Domain Model
- HEALING_DOMAIN: self-reflection, journaling, emotional regulation, psychoeducation, safety routing.
- BUSINESS_DOMAIN: SEO, publishing, analytics, monetization, internal growth operations.
- PLATFORM_DOMAIN: runtime, routes, APIs, auth, database, billing, CI/CD, deployment, observability.
- DESIGN_DOMAIN: UI, visual system, tokens, Lumi avatar, accessibility, motion.
- RESEARCH_DOMAIN: evidence synthesis, citations, uncertainty separation.
- CROSS_DOMAIN: split into isolated subplans with boundaries.

## Business / Healing Firewall
Business systems may use only aggregate, privacy-safe operational data.
Healing systems must never use pricing pressure, vulnerability targeting, dark patterns, or coercive retention logic.

## Replit Runtime Law
Before feature work, verify:
- .replit
- package.json
- active entrypoint
- process.env.PORT
- app.listen()
- host binding
- health endpoint
- ready endpoint
- duplicate runtime files
- build/start command alignment

## Avatar + Visual Law
Lumi is a product companion system, not decoration.
Lumi must optimize for:
- warmth
- calm
- safety
- trust
- gentle motion
- emotional clarity
- official brand consistency
- reduced-motion accessibility

Canonical palette:
- Serenity Sage: #4A7E72
- Eternal Gold: #B8963E
- Rose Blush: #D4857A
- Warm Cream: #E8D5A0

## Completion Rule
A recommendation is not complete until it has:
- file evidence
- gate evidence
- route/build evidence where relevant
- rollback path
- phase summary
