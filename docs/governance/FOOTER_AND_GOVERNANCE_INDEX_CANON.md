# Phase 114 — Footer + Governance Index Canon

## Purpose
Create the canonical plan for simplifying the public footer and exposing governance documents safely.

## Core Law
The footer must reduce cognitive load, not expand it.

## Canonical Footer Columns
1. Healing
2. Learn
3. Trust
4. About
5. Crisis

## Required Footer Rules
- No dense link walls.
- No manipulative CTAs.
- No urgency language.
- Crisis access must remain visible.
- Trust + AI Transparency must be easy to find.
- Business links must not interrupt healing flows.

## Governance Index Goal
Create a public, read-only governance index later from approved docs only.

## Forbidden
- Do not expose private logs.
- Do not expose internal secrets.
- Do not expose user data.
- Do not expose operational vulnerabilities.
- Do not mix healing content with monetization funnels.

## Future Source Implementation
Later implementation may add:
- /governance
- simplified footer groups
- generated governance doc index
- Trust Center reciprocal footer links
- AI Transparency reciprocal footer links
