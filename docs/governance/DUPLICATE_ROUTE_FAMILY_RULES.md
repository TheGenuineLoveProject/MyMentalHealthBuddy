# Duplicate Route Family Rules

## Purpose
Prevent unsafe deletion or refactor of route families.

## Rules
1. Do not delete routes based only on similar names.
2. Identify canonical owner first.
3. Classify duplicates as: keep, alias, redirect, archive, future, or unsafe-to-touch.
4. Crisis, auth, billing, admin, safety, privacy, and legal routes require protected review.
5. This phase is documentation only.
