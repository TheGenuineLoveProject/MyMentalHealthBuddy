# Routing Governance

## Rules

1. One canonical route owner per feature domain
2. No duplicate emotional systems
3. No duplicate onboarding flows
4. No duplicate dashboard systems
5. Experimental routes must be prefixed or isolated
6. Admin systems must remain access-controlled
7. Crisis systems must remain globally accessible
8. Production routes must remain emotionally safe
9. No manipulative retention loops
10. No runtime dependency on experimental modules
