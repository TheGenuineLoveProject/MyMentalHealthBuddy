# INTERACTION DOMAIN GOVERNANCE

## PURPOSE

Define strict interaction boundaries between:

- HEALING_DOMAIN
- BUSINESS_DOMAIN
- PLATFORM_DOMAIN
- SAFETY_DOMAIN
- CROSS_DOMAIN

This governance layer prevents:
- emotional monetization
- manipulative interaction sequencing
- vulnerable-state conversion pressure
- unsafe emotional-state transitions
- monetization contamination of healing flows

---

| Action Type | Domain |
|---|---|
| Navigation | PLATFORM_DOMAIN |
| Reflection | HEALING_DOMAIN |
| Meditation | HEALING_DOMAIN |
| Journaling | HEALING_DOMAIN |
| Mood Tracking | HEALING_DOMAIN |
| Cognitive Analysis | CROSS_DOMAIN |
| Atlas Routing | CROSS_DOMAIN |

| Billing | BUSINESS_DOMAIN |
| Paywall Tracking | BUSINESS_DOMAIN |
| Crisis Links | SAFETY_DOMAIN |
| Escalation Routing | SAFETY_DOMAIN |

---

# PRIMARY GOVERNANCE LAW

BUSINESS ACTIONS MUST NEVER EXECUTE INSIDE REGULATED HEALING FLOWS.

Meaning:

PROHIBITED:

- paywall popups during meditation
- upsells after vulnerability disclosures
- conversion pressure after journaling
- pricing prompts during reflection
- emotional-state-triggered monetization
- monetization after crisis disclosures
- adaptive pricing based on emotional state
- emotional vulnerability scoring
- emotional manipulation loops
- psychologically-timed conversion prompts

---

# ATLAS GOVERNANCE

Atlas is classified as:

CANONICAL COGNITIVE NAVIGATION LAYER

Atlas responsibilities:

- pathway routing
- tool discovery
- reflective sequencing
- cognitive organization
- structured navigation
- regulated transitions
- state-aware routing

Atlas MUST NOT:

- inject emotional monetization
- trigger conversion pressure
- infer vulnerability for monetization
- alter pricing based on emotional state
- introduce urgency manipulation
- create dependency loops

---

# HEALING FLOW PROTECTION RULES

Inside HEALING_DOMAIN:

ALLOWED:
- calming UI
- grounding tools
- journaling
- rituals
- reflection
- educational wellness support
- emotional regulation support

PROHIBITED:
- subscription interruptions
- upgrade pressure
- countdown timers
- scarcity prompts
- emotional urgency monetization
- emotionally-timed upsells
- conversion interruption overlays

---

# SAFETY DOMAIN PRIORITY

SAFETY_DOMAIN overrides all domains.

If:
- crisis indicators
- escalation states
- dysregulation markers
- acute distress indicators

are detected:

BUSINESS_DOMAIN actions must suspend.

---

# CROSS DOMAIN RULES

CROSS_DOMAIN systems must:
- isolate analytics
- avoid emotional monetization
- preserve privacy boundaries
- avoid behavioral exploitation
- preserve emotional safety
