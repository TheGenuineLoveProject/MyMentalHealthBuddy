# Semantic Search + Visible Tool Index Governance

## Purpose
Create the canonical governance layer for user-friendly discoverability, emotional search, tool indexing, and journey routing.

## Core Law
Users should be able to find the right support by:
- what they feel
- what they need
- what goal they have
- what tool category fits
- where they are in the journey

## Search Modes
1. Emotional search
2. Tool search
3. Topic search
4. Journey search
5. Safety/trust search
6. Resource search

## Required Metadata Per Tool/Page
- title
- slug
- route
- category
- emotionalState
- journeyStage
- userIntent
- safetyLevel
- relatedTools
- relatedTopics
- canonicalOwner

## Forbidden
- manipulative recommendations
- crisis misrouting
- healing-data monetization
- hidden vulnerability scoring
- duplicate tool indexes
- search results without safety boundaries

## Quality Gates
- build passes
- production health passes
- no source edits in this phase
- registry exists
- visible index contract exists
- routes remain unchanged
