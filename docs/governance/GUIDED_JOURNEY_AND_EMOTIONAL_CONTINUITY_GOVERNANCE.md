# Guided Journey + Emotional Continuity Governance

## Purpose
Create a governed structure for emotional journeys without touching source code.

## Core Law
Users must feel oriented, safe, and gently guided — never manipulated, pressured, diagnosed, or emotionally exploited.

## Primary Journey Types
1. Anxiety grounding journey
2. Burnout recovery journey
3. Sleep restoration journey
4. Self-worth journey
5. Grief support journey
6. Emotional overwhelm journey
7. Calm reset journey
8. Reflection and journaling journey
9. Relationship healing journey
10. Daily wellness routine journey

## Required Journey Pattern
Each journey must define:
- user emotional state
- safest first step
- recommended tool sequence
- crisis boundary
- non-clinical disclaimer
- completion signal
- next gentle option

## Forbidden
- no diagnosis
- no suicidality determination
- no coercive retention
- no shame language
- no fear-based urgency
- no healing-data monetization
- no addictive streak pressure
- no hidden persuasion

## Quality Gates
Every journey must pass:
- safety review
- privacy review
- route review
- accessibility review
- mobile UX review
- content coherence review
- rollback readiness
