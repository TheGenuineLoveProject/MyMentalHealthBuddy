# Canonical Information Architecture Law

## Purpose

Establish one canonical discoverability and navigation system.

Prevent:
- route sprawl
- semantic duplication
- taxonomy drift
- orphan pages
- conflicting navigation
- inconsistent indexing
- discoverability fragmentation

## Governance Rules

1. Every route must belong to ONE canonical semantic hub.

2. Every route must contain:
- title
- description
- discoverability category
- canonical navigation owner
- indexing status

3. Duplicate semantic routes are prohibited.

4. Navigation ownership must be singular.

5. Every page must be internally reachable.

6. Orphan routes are prohibited.

7. All discoverability systems must derive from:
registry/information-architecture/canonical-ia-registry.json

8. No expansion before validation.
