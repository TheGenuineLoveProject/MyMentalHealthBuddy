# Emotional State UX Audit Governance

## Purpose
Create a governance-first audit system for emotionally-safe UX.

## Core Principle
Every interface affects nervous-system state.

UX must:
- reduce overwhelm
- increase clarity
- preserve calm
- support reflection
- minimize cognitive overload
- reduce emotional friction

## Audit Categories

### Calmness
- visual pacing
- whitespace
- typography comfort
- transition softness

### Cognitive Load
- too many choices
- excessive density
- fragmented hierarchy
- navigation ambiguity

### Emotional Safety
- shame language
- urgency pressure
- manipulative framing
- overstimulation

### Accessibility
- reduced motion
- readable contrast
- keyboard navigation
- screen reader support

### Continuity
- emotional continuity
- semantic continuity
- journey continuity
- orientation clarity

## Forbidden UX Patterns
- infinite urgency
- manipulative countdowns
- addictive streak pressure
- emotional dependency framing
- fear-driven retention
- aggressive interruption patterns

## Quality Gates
- build verification
- health verification
- accessibility verification
- rollback readiness
- no protected surface changes
