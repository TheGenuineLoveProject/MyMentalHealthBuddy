# Canonical Route Registry Law

## Purpose
Establish a single source of truth for route governance.

## Goals
- prevent taxonomy drift
- prevent duplicate trust surfaces
- preserve discoverability coherence
- preserve admin isolation
- preserve canonical wellness routing

## Laws

### 1. Trust Law
Trust routes must never silently shadow each other.

### 2. Discoverability Law
Discovery routes may redirect to canonical wellness surfaces.

### 3. Alias Law
Aliases must be documented in registry governance.

### 4. Admin Isolation Law
Admin routes must remain isolated from public wellness bundles.

### 5. Governance Law
Duplicate routes require governance review before removal.
