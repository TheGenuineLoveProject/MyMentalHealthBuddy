# Route Ownership Registry

## Purpose
Define which route/page family owns each major platform experience.

## Production Core
- Home / landing
- Login / account
- Dashboard
- Journal
- Mood tracking
- AI companion
- Crisis resources
- Privacy / legal / safety
- Billing / subscription
- Settings

## Admin Core
- Admin dashboard
- Security dashboard
- Publishing dashboard
- Audit dashboard
- Analytics dashboard
- User management

## Experimental / Controlled
- Lumi modules
- Ritual systems
- Scene systems
- Meta-learning systems
- Ontology / philosophy / paradox systems
- Advanced avatar labs

## Archive / Backup
- .hx-backups
- old snapshots
- prior generated diagnostics
- duplicate experimental trees

## Rule
Every future route must have exactly one owner tier:
PRODUCTION, ADMIN, EXPERIMENTAL, ARCHIVE, or FUTURE.
