# Canonical Feature Tier Governance

## TIER 1 — PRODUCTION CORE
Stable user-facing systems required for production operation.

Examples:
- authentication
- onboarding
- dashboard
- journal
- mood tracking
- AI companion
- crisis resources
- wellness tools
- settings
- legal/privacy
- billing
- analytics

---

## TIER 2 — CONTROLLED EXPERIMENTAL
Advanced systems under observation.

Examples:
- lumi-scenes
- lumi-memory
- lumi-consistency
- ontology systems
- paradox systems
- meta-learning systems
- ritual systems

These must remain:
- isolated
- optional
- non-blocking
- rollback-safe

---

## TIER 3 — INTERNAL ADMIN
Operational systems.

Examples:
- admin dashboards
- audit systems
- publishing systems
- moderation systems
- analytics admin
- feature flags

---

## TIER 4 — ARCHIVAL
Historical or deprecated systems.

Rules:
- no active routing
- no runtime dependency
- excluded from production bundle evolution

---

## TIER 5 — FUTURE STATE
Research-stage concepts not approved for production.
