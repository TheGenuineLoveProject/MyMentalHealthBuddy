# Phase 40 — Security Audit

Purpose:
- Enumerate current vulnerabilities.
- Determine safe upgrade path.
- Preserve deployment stability.

Rules:
- No force upgrades.
- No dependency removals.
- No refactors.
- No runtime architecture changes.
- No auth/database/infrastructure changes.

Required outputs:
- Vulnerability inventory
- Runtime risk classification
- Safe upgrade candidates
- Unsafe upgrade candidates
- Recommended patch sequence
