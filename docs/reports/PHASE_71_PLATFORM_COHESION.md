# Phase 71 — Canonical Tool Directory

## Purpose
Improve platform coherence and discoverability by creating a unified wellness tool universe.

## Problem
Tools existed but were fragmented across:
- hubs
- pages
- trackers
- rituals
- reflections
- pathways
- dashboards

This reduced discoverability and weakened perceived platform depth.

## Solution
Introduce:
- canonical tool registry
- searchable directory
- category system
- emotional-goal taxonomy
- visibility classification
- coherence layer

## Runtime Risk
Low.

No:
- auth changes
- billing changes
- crisis changes
- route deletions
- database mutations

## Expected Outcome
Users can:
- discover tools,
- understand platform structure,
- navigate coherently,
- perceive true platform depth.
