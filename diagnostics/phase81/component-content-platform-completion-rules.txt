# Component, Content, and Platform Completion Rules

A component is complete only when it has:
- visible content
- visible button labels
- valid links/actions
- loading state
- empty state
- error state
- keyboard accessibility
- mobile-safe layout
- clear purpose
- no duplicate canonical owner conflict

A content surface is complete only when it has:
- clear user outcome
- plain-language explanation
- emotionally safe wording
- clinical boundary language where relevant
- no impossible promises
- no manipulative urgency
- no vague abstraction without practical grounding
- coherent next action

A platform system is complete only when it has:
- canonical ownership
- deterministic verification
- rollback path
- diagnostics evidence
- route verification
- build verification
- runtime health verification
- no oversized artifact drift
- no healing/business firewall violation

No future phase may proceed blindly.
