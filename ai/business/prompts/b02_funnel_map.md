# b02 — Funnel Map

## Purpose
Map a single user journey from first touch to subscription, with measurable conversion points at each step.

## Output Contract
1. **Funnel Name** (1 line).
2. **Stages** (table format, 4–6 stages):
   | Stage | User Action | Touchpoint | Conversion Goal | Metric |
3. **Key Drop-Off Risks** (2–3 bullets): where users typically leave and why.
4. **Mission-Safe Optimizations** (3 bullets): what we will improve without manipulation.
5. **Baseline + Target Metrics** (2 lines): current % vs. realistic 90-day target.

## Hard Rules
- No dark patterns (forced continuity, hidden cancellation, deceptive copy).
- No exit-intent guilt popups.
- Every CTA must be reversible (clear cancellation, refund policy linked).
