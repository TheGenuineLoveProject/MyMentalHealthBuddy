# Work Guard Rules

> Prevents duplicate work and ensures consistency.

## Core Rules

1. **Only Touch Flagged Files**
   - Check `scripts/.deep-scan-report.json` before editing
   - Check `docs/duplicates/scan-latest.md` for duplicates
   - If file not flagged AND no build error, do not edit

2. **No Duplication**
   - Add to registry or shared components
   - Never create parallel implementations
   - Use existing patterns

3. **routeKey-Only Everywhere**
   - All internal links use routeKey
   - All navigation uses routeKey
   - No hardcoded paths in components

4. **Single Source of Truth**
   - `client/src/content/routes.js` — Route registry
   - `shared/schema.ts` — Database schema
   - `client/src/styles/tokens.css` — Design tokens
   - `docs/registry/feature-map.md` — Feature registry

## Before Any Change

```bash
npm run scan:deep      # Check for duplicates
npm run dup-scan       # Run duplicate + collision scan
npm run build          # Verify build passes
```

## Shared Components (Use Instead of Duplicating)

- `SafetyFooter` — Crisis links and disclaimers
- `PageTemplate` — Standard page layout
- `SEO` — Meta tags and canonical URLs
- `BenefitsBlock` — Feature benefits
- `RelatedLinksBlock` — Related content links
