# Dynamic Reflection Content — UI Integration

This document defines how reflection content is
loaded dynamically into the platform UI.

## CONTENT SOURCE

Primary Source:
• /marketing/content/posts-batch-01.md

Future Sources:

---

## DATA MODEL (CONCEPTUAL)

Each reflection item contains:
• id
• pillar (reflection / education / compassion)
• text
• optional prompt
• created_at

No engagement metrics.
No popularity sorting.

---

## UI BEHAVIOR

### Reflection Feed
• Randomized gentle ordering
• No “best” or “trending”
• No counts

### Reflection View
• One reflection at a time
• Clear spacing
• Calm typography
• Optional journaling input

### Completion State
• Gentle closing message
• No CTA beyond rest or reflection

---

## USER AGENCY RULES

• User may exit at any time
• No forced progression
• No time pressure

---

## BRAND ENFORCEMENT

• Colors from colors.md
• Fonts from typography.md
• Spacing favors calm
• No visual noise

---

## RESULT

Reflection content feels alive
without becoming performative.
