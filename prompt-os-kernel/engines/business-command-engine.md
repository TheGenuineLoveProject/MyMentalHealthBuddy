# Business Command Engine

## Purpose
Structure all BUSINESS_DOMAIN outputs into actionable, measurable, auditable formats.

## Output Contract
Every business output must include:
1. Hard Truth
2. Decision Context
3. Core Issue
4. Root Cause
5. Strategic Priority
6. Immediate Actions
7. Medium-Term Actions
8. Risks
9. Metrics to Watch
10. Decision Challenge

## Available Engines
- Healing Engine — HEALING_DOMAIN only
- Research Engine — RESEARCH_DOMAIN
- Strategy Engine — BUSINESS_DOMAIN strategy
- Engineering Engine — PLATFORM_DOMAIN
- Design Engine — DESIGN_DOMAIN
- Growth Engine — user acquisition and retention
- Monetization Engine — pricing, conversion, revenue
- Automation Engine — workflow automation
- Analytics Engine — metrics, dashboards, observability
- Security Engine — auth, encryption, compliance
- Governance Engine — policy enforcement
- Prompt Evolution Engine — prompt lifecycle management
- Clarity Engine — communication and UX clarity
- Publishing Engine — content publishing pipeline
- SEO Engine — search optimization
- Brand Integrity Engine — brand consistency enforcement

## Rule
Use the smallest valid engine set required for each task.

## SEO + Publishing Output
When the task involves SEO, blog, content, newsletter, or social:
- objective, audience, search intent, content angle
- topic cluster, asset plan, distribution plan
- CTA plan, measurement plan, compliance/safety checks

## SEO Rules
- No keyword stuffing
- No fabricated authority
- No manipulative mental health claims
- Separate education from clinical claims
- Prefer topic clusters and semantic relevance
- Preserve brand tone and trust
