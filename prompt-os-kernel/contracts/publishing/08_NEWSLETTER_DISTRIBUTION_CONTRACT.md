# B8 — NEWSLETTER DISTRIBUTION CONTRACT

## Contract ID
NEWSLETTER_DISTRIBUTION_CONTRACT

## Domain
BUSINESS_DOMAIN

## Purpose
Define newsletter-ready content requirements, audience gating, and safe distribution boundaries. This contract governs how published content is adapted and distributed via email newsletters through the Resend integration.

## Newsletter Eligibility

Content is eligible for newsletter distribution if ALL conditions are met:
1. Status is `published`
2. Visibility is `public` or `subscribers`
3. Content type is `blog`, `newsletter`, `essay`, or `reflection`
4. Content has passed all B7 QA checks
5. Author has not marked content as `newsletter_exclude`

## Newsletter Content Requirements

| Field | Source | Required |
|-------|--------|----------|
| subject | Content title or custom newsletter subject | yes |
| preheader | Summary truncated to 100 characters | yes |
| body | Content body adapted for email (see adaptation rules) | yes |
| cta_link | canonicalUrl to full content on site | yes |
| cta_text | Call-to-action text, e.g., "Read the full article" | yes |
| unsubscribe_link | Auto-generated by Resend | yes |
| sender_name | "The Genuine Love Project" | yes |
| sender_email | Configured Resend sender address | yes |

## Content Adaptation Rules

1. Strip all interactive elements (forms, embedded tools, JavaScript)
2. Convert markdown to email-safe HTML
3. Limit body to 500 words with a "continue reading" link
4. Preserve all images with absolute URLs
5. Remove internal navigation links
6. Add plain-text fallback version

## Audience Gating

| Audience | Description | Criteria |
|----------|-------------|----------|
| all_subscribers | All email subscribers | Default |
| plan_subscribers | Users with active paid plan | subscription_status != free |
| segment | Custom segment by tag or behavior | Defined per campaign |

## Domain Safety Rules

1. HEALING_DOMAIN content distributed via newsletter must not include monetization CTAs
2. Newsletter CTAs for healing content should link to the content only, not pricing pages
3. Every newsletter must include an unsubscribe link
4. No manipulative urgency language ("Last chance!", "Don't miss out!")
5. Subject lines must accurately represent content, no clickbait

## Frequency Rules

1. Maximum one newsletter per day to any single subscriber
2. Maximum three newsletters per week to any single subscriber
3. Subscribers may set frequency preferences (daily, weekly, monthly)
4. Respect subscriber preferences over campaign schedule

## Distribution Metadata (B1 Reference)

The `distribution.newsletter` object shape:
```
{
  "eligible": boolean,
  "sent": boolean,
  "sentAt": ISO 8601 | null,
  "audience": "all_subscribers" | "plan_subscribers" | "segment",
  "segmentId": string | null,
  "subject": string | null,
  "campaignId": string | null
}
```
