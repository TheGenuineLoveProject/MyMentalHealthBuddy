## EXECUTION PIPELINE (every cycle, in this exact order)

### PHASE 0 — INSPECT
Run diagnostics only. Report current state. **Do NOT patch anything.**

### PHASE 1 — CLASSIFY
Categorize Phase 0 findings into: ACTIVE / BROKEN / DUPLICATE / DEAD / HIGH-RISK.

### PHASE 2 — IDENTIFY ONE BLOCKER
Pick **ONE** verified blocker. Not ten. Not five. ONE. Smallest safe patch.

### PHASE 3 — PATCH
Fix only that ONE blocker. **No side-quests. No "while I'm here…"**

### PHASE 4 — VERIFY (six gates)
- `BUILD_GATE` — `npm run build` passes
- `ROUTE_GATE` — all routes resolve
- `HEALTH_GATE` — `/health`, `/ready` respond 200
- `TSC_GATE` — `npx tsc --noEmit` 0 errors
- `ROLLBACK_GATE` — rollback path documented
- `DUPLICATION_GATE` — no new duplicates created

### PHASE 5 — DOCUMENT
Log change in `replit.md` as ONE LINE. Move detail to `docs/changelog.md`. STOP.

### PHASE 6 — STOP
Do not proceed until user confirms. Next cycle starts from Phase 0.
